# 健康办公

## 后续迭代规则

- 默认只做浏览器 Web 适配；Electron / 桌面端只在明确提出时处理。
- 默认采用快速模式：小迭代优先跑 `npm run verify:quick`，不默认跑全量 smoke 或桌面打包。
- 每轮需求先锁定影响范围，再读取对应模块；功能索引见 `docs/PROJECT_MAP.md`。
- 只有明确说“准备验收”或“准备交付”时，才跑 `npm run verify:full` 和桌面端相关验证。
- 模块化按需求逐步推进，不做一次性过度拆分。

健康办公是一个面向办公场景的健康微干预 Demo，围绕“状态感知 -> 风险触发 -> 即时提醒 -> 动作引导”形成闭环。项目使用 `React + TypeScript + Vite` 构建，并已接入 Electron，可打包为 Windows 免安装桌面应用。

## 当前功能

- 健康用眼
  - 连续用眼提醒
  - 视距检测：过近 / 正常 / 偏远
  - 视距校准：按当前舒适距离采样并持久化到本地
  - 低眨眼频率提醒
  - 今日眨眼计数
- 姿态检测
  - 低头
  - 前倾
  - 歪头
  - 肩线倾斜
  - 姿态基线校准：按当前自然工作坐姿采样并持久化到本地
- 脸屏适配
  - 支持单屏模式 / 多屏模式选择
  - 单屏模式页 / 多屏模式页共用同一套实时感知窗口与提醒能力
  - 单屏或多屏完成配置与校准后，才允许开始真实办公监测
  - 多屏适配以弹窗方式配置，3D 画布使用懒加载 Three.js；画布内可添加、删除、校准屏幕，拖右侧控制杆查看空间盒子
  - 优先使用 Face Landmarker Worker 做 3D 头姿估计
  - 不可用时自动回退到 2D 代理
  - 根据脸与屏幕的相对位置自动适配视距与姿态判断
- 统一提醒
  - 顶部轻量 Toast
  - 支持手动完成
  - 支持连续眨眼 3 次关闭
- Demo 模式
  - 可快速模拟连续盯屏、距离过近、低头、前倾、歪头、肩线倾斜、低眨眼频率
- 调试 HUD
  - 可显示 / 隐藏
  - 支持拖动
  - 展示摄像头状态、视距、姿态、眨眼、用眼在岗、连续用眼等核心信号
- 设置
  - 作为单屏 / 多屏模式共用入口
  - 点击“设置”后以弹窗展示提醒基准、姿态提醒开关和视距阈值

## 当前不包含

- 饮水提醒
- 久坐提醒
- 眨眼频率趋势图与导出
- 动眼引导 / gaze 游戏
- 驼背提醒

## 安装依赖

```bash
npm install
```

如果首次安装 Electron 较慢，可先设置镜像：

```powershell
$env:ELECTRON_MIRROR='https://npmmirror.com/mirrors/electron/'
npm install
```

## 浏览器开发模式

```bash
npm run dev
```

默认地址：

```text
http://127.0.0.1:5173/
```

## 桌面应用运行

本地直接运行桌面版：

```bash
npm run desktop:run
```

这会先构建前端，再启动 Electron 独立窗口。桌面版不依赖外部浏览器。



## 摄像头权限

- 浏览器模式：点击页面内“允许摄像头”，并在浏览器弹窗中授权
- 桌面模式：点击“允许摄像头”后，Electron 会请求系统摄像头权限
- 如果之前拒绝过，需要在系统隐私设置中重新开启权限

## 使用建议

1. 先选择“单屏模式”或“多屏模式”
2. 单屏模式：点击“允许摄像头”，完成“视距和姿态校准”，再点击“开始办公”
3. 多屏模式：先打开“多屏配置”弹窗，在 Three.js 画布中添加屏幕、编辑屏幕名称、设置摄像头屏、拖动屏幕位置，并用底部控制条调整前后 Z、左右夹角、上下倾角和尺寸；需要换视角时拖右侧控制杆，可在画布内重置视角，再完成摄像头授权与校准
4. 点击“设置”可打开通用设置弹窗，调整用眼提醒、姿态提醒开关和视距阈值
5. 如需演示，开启 Demo 模式并使用快捷触发

## 提醒规则总览

### 功能规则表

| 功能 | 检测条件 | 触发门槛 | 提醒时机 | 优先级 |
|---|---|---|---|---:|
| 距离过近 | `视距=too-close` | 持续 `90 秒`，Demo `8 秒` | 单独提醒 | 90 |
| 低头 | `姿态=head-down` 且开关开启 | 持续 `75 秒`，Demo `8 秒` | 单独提醒 | 92 |
| 前倾 | `姿态=forward-head` 且开关开启 | 持续 `75 秒`，Demo `8 秒` | 单独提醒 | 89 |
| 歪头 | `姿态=head-tilt` 且开关开启 | 持续 `75 秒`，Demo `8 秒` | 单独提醒 | 82 |
| 肩线倾斜 | `姿态=shoulder-tilt` 且开关开启 | 持续 `75 秒`，Demo `8 秒` | 单独提醒 | 84 |
| 低眨眼频率 | `眨眼状态=low-rate` | 持续 `25 秒`，Demo `8 秒` | 单独提醒 | 76 |
| 连续用眼过久 | 连续用眼累计达到阈值 | `20 分钟`，Demo `20 秒` | 单独提醒 | 74 |
| 组合风险 | `距离过近` + `低头/前倾` | 距离与姿态都各自持续达标 | 组合高优先级提醒 | 100 |

补充说明：

- `持续达标` 指某个异常状态先被系统稳定识别出来，再连续保持超过设定时长，才进入提醒候选。
- `用眼在岗` 需要先确认人脸稳定存在，真实模式约 `2.5 秒`，Demo 模式约 `1 秒`。

### 用户可见文案表

| 提醒项 | 主建议 | 原因说明 | 关闭方式 |
|---|---|---|---|
| 组合风险 | 后移一点并坐正 | 因为离屏幕过近，而且姿态也偏了 | 完成 / 连续眨眼 3 次 |
| 距离过近 | 后移一点并坐直 | 因为离屏幕太近了 | 完成 / 连续眨眼 3 次 |
| 低头 | 抬头看向屏幕上沿 | 因为低头时间已经有点久了 | 完成 / 连续眨眼 3 次 |
| 前倾 | 轻收下巴并后移 | 因为头部前倾了 | 完成 / 连续眨眼 3 次 |
| 歪头 | 把头轻轻扶正 | 因为头部持续偏向一侧 | 完成 / 连续眨眼 3 次 |
| 肩线倾斜 | 放平肩线并放松肩颈 | 因为一侧肩膀明显更高 | 完成 / 连续眨眼 3 次 |
| 低眨眼频率 | 连续轻眨几次 | 因为最近眨眼明显偏少 | 完成 / 连续眨眼 3 次 |
| 连续用眼过久 | 看向远处 20 秒 | 因为连续盯屏时间有点久了 | 完成 / 连续眨眼 3 次 |

### 调试 / 内部逻辑表

| 项目 | 当前逻辑 |
|---|---|
| 提醒候选生成 | 根据视距、姿态、眨眼、连续用眼状态生成提醒候选 |
| 统一调度 | 同一时刻只保留 1 个主提醒 |
| 选择规则 | 先按优先级，再按谁更早到期 |
| 分组冷却 | 同组提醒处理后进入冷却，避免短时间重复弹出 |
| 全局静默 | 刚处理完一个提醒后，整体短时间不再弹新提醒 |
| 真实模式冷却 | 组冷却 `6 分钟`，全局静默 `75 秒` |
| Demo 模式冷却 | 组冷却 `16 秒`，全局静默 `8 秒` |
| 自动关闭 | 当对应异常条件消失后，提醒会自动收起 |
| 手动关闭 | 点击 `完成` |
| 眨眼关闭 | 连续眨眼 3 次 |
| 姿态开关 | 只影响“是否提醒”，不影响姿态检测本身 |
| Demo 模式 | 直接覆盖实时状态，但仍走同一套提醒候选与调度逻辑 |
| 当前不建议开放 | 用户自定义提醒优先级；因为它和组合提醒、分组冷却强耦合 |

如果需要人工验证侧屏、姿态或眨眼表现，请按 [健康办公真实环境测试流程](docs/real-world-test-flow.md) 执行。测试时可打开 HUD 并点击“复制快照”，把快照和问题编号一起反馈，便于快速定位。

## 已知限制

- 视距、姿态、眨眼检测都是前端近似判断，不是医疗诊断
- 识别效果会受到光照、摄像头角度、眼镜反光、入镜完整度影响
- 3D 头姿会优先使用 Face Landmarker Worker，不可靠时自动回退到 2D 代理
- Windows portable 包未做代码签名，首次运行时系统可能提示安全确认
- Electron 包体积较大，这是内置 Chromium、模型资源和本地运行时带来的正常代价

## 非医疗诊断说明

本项目仅用于办公健康场景下的轻提醒与演示，不构成医疗诊断、治疗建议或专业康复评估。
