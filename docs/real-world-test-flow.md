# 健康办公真实环境测试流程

这份流程用于你做真实摄像头、多屏、姿态和眨眼测试后，把结果快速反馈给 Codex 继续优化。

## 测试前准备

1. 启动项目：`npm run dev`
2. 打开 `http://127.0.0.1:5173/`
3. 选择屏幕模式：单屏模式或多屏模式
4. 点击“允许摄像头”
5. 如测试多屏场景，先打开“多屏配置”弹窗，通过 Three.js 空间盒子画布确认摄像头所在屏和工作屏的 3D 位置关系
6. 点击“视距和姿态校准”
7. 校准完成后点击“开始办公”
8. 打开“显示 HUD”

## 如何清空本地数据

在浏览器控制台执行：

```js
localStorage.clear()
location.reload()
```

这会清空视距校准、姿态校准、多屏配置、设置项和 Demo 状态。只在需要测试“首次使用/无本地数据”时执行。

## 如何反馈调试快照

1. 打开“显示 HUD”
2. 点击 HUD 顶部“复制快照”
3. 把复制出来的内容粘贴给 Codex
4. 再补充你的体感判断，例如“我实际是坐直的，但提示前倾”

## P0 测试卡片：侧屏办公

请按下面编号测试，只记录问题即可；正常项可以写“没有可见问题”。

1. 打开多屏配置弹窗：确认能看到完整空间盒子网格，并可以拖动右侧控制杆旋转视角。
2. 配置完成后关闭弹窗：确认实时感知窗口没有丢失摄像头画面。
3. 拖动右侧控制杆旋转空间盒子：确认可以从不同视角看清屏幕前后位置关系，并确认靠近用户为 -Z。
4. 点击画布内“添加屏幕”：确认新屏幕默认正对用户、无左右夹角和上下倾角；最多只能添加到 8 块总屏幕。
5. 点击画布内“重置视角 / 设为摄像头屏 / 校准屏幕 / 删除屏幕”：确认能直接对选中屏幕操作。
6. 拖动底部控制条：确认前后 Z、左右夹角、上下倾角、尺寸按顺序排列，且尺寸不会无限放大。
7. 切换不同屏幕：确认右侧面板能看到每块屏幕的校准状态，画布内能编辑选中屏幕名称，并能看到空间、姿态、尺寸、校准信息。
8. 正对摄像头屏办公：观察当前工作屏、视距状态、姿态状态是否稳定。
9. 头部不移动，只把目光/脸转向左侧工作屏：观察当前工作屏是否切换，视距是否仍接近正常。
10. 头部不移动，只把目光/脸转向右侧工作屏：观察当前工作屏是否切换，视距是否仍接近正常。
11. 身体整体转向左侧工作屏：观察是否误报歪头、前倾或距离过近。
12. 身体整体转向右侧工作屏：观察是否误报歪头、前倾或距离过近。
13. 在侧屏状态下低头：观察是否能触发低头。
14. 在侧屏状态下前倾：观察是否能触发前倾。
15. 在侧屏状态下歪头：观察是否能触发歪头。
16. 在侧屏状态下耸左肩/右肩：观察肩线倾斜两侧灵敏度是否接近。
17. 在近距离快速晃头但不眨眼：观察今日眨眼是否误增加。
18. 在较远距离自然眨眼：观察今日眨眼是否能正常增加。

## 注意事项

- 视距、姿态、眨眼都是前端近似健康判断，不是医疗诊断。
- 侧屏场景建议每次测试都附带一次 HUD 快照，尤其是出现误报时。
- 如果切换窗口后出现状态异常，请记录“切换前状态、切换后状态、等待多久仍未恢复”。
